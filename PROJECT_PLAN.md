# Omni_SOC 项目完整技术架构与开发规划

## 📋 项目概览

**项目名称**: Omni_SOC (AI-Driven Autonomous Security Operations Center)
**当前版本**: V1.0 (原型阶段)
**核心目标**: 基于Agentic工作流构建下一代自动化安全运营平台
**技术栈**: Python 3.10+, LangGraph, LLM (智谱GLM-4), LangChain

---

## 🏗️ 技术架构设计

### 核心设计理念

**节点设计原则**: 节点代表事件管理的生命周期阶段，而非具体技术实现
- 每个节点对应安全分析师的决策阶段
- 具体技术实现作为工具供智能体自主选择
- 确保业务流程清晰、可追溯

### 2. 核心组件架构

#### 2.1 状态管理系统 (OmniState)
```python
OmniState:
  - raw_alert: 原始告警数据
  - evidence_pool: 证据池 (可追加列表)
  - thought_log: 思考日志 (可追加列表)
  - risk_score: 风险评分 (0-100)
  - status: 处理状态
  - next_action: 下一步动作
  - final_report: 最终报告
  - context: 上下文信息
  - metadata: 元数据
```

#### 2.2 工作流引擎
- **状态机模式**: 基于LangGraph的有向图执行
- **条件路由**: 根据状态动态决策执行路径
- **并行执行**: 支持多个节点并行工作
- **回滚机制**: 支持状态回滚和重试

#### 2.3 技能系统 (Skills)
- **进程分析**: 进程树构建、进程行为分析
- **网络分析**: 连接分析、DNS查询、流量特征分析
- **文件分析**: 静态分析、动态分析、威胁检测
- **威胁情报**: 威胁情报查询、信誉检查、IOC验证

